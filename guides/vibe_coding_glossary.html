웹의 기본 구조6개 용어
1
HTMLHyperText Markup Language
꼭 알아야
▶
웹페이지의 뼈대입니다. "여기는 제목", "여기는 문단", "여기는 버튼"처럼 내용의 구조를 정하는 언어입니다.
우리가 만들 홈페이지의 모든 텍스트, 이미지, 버튼, 표가 HTML로 정의됩니다.
매장의 평면도(도면)와 같습니다. "여기가 주방, 여기가 홀, 여기가 계산대"처럼 공간을 나누는 설계도. 아직 인테리어(색상, 디자인)는 없는 상태입니다.
실제 코드 예시
<!-- 이것이 HTML입니다 --> <h1>제2판교 트라이타워 입점 검토</h1> <p>종합 점수: 67점</p> <button>분석 실행</button>
<h1>은 제목, <p>는 문단, <button>은 버튼을 뜻합니다. 이런 것들을 태그(tag)라고 부릅니다.
관련 용어: CSS, 태그
2
CSSCascading Style Sheets
꼭 알아야
▶
웹페이지의 디자인(외관)을 담당합니다. 색상, 크기, 글꼴, 간격, 레이아웃 등 눈에 보이는 모든 것을 CSS로 꾸밉니다.
HTML이 뼈대라면, CSS는 인테리어입니다.
매장의 인테리어 디자인입니다. 벽 색상은 뭘로 할지, 테이블 간격은 얼마나 둘지, 간판 글씨 크기는 얼마로 할지. 같은 평면도(HTML)라도 인테리어(CSS)에 따라 분위기가 완전히 달라집니다.
실제 코드 예시
/* pfchangs 대시보드의 CSS 일부 */ background: #f0efec; /* 배경색: 따뜻한 오프화이트 */ color: #378ADD; /* 글자색: 파란 액센트 */ border-radius: 12px; /* 모서리 둥글기 */ font-size: 13px; /* 글자 크기 */
관련 용어: HTML, 반응형
3
JavaScript (JS)자바스크립트
꼭 알아야
▶
웹페이지의 동작과 기능을 담당합니다. 버튼을 클릭하면 차트가 바뀌고, 슬라이더를 움직이면 숫자가 재계산되는 것이 모두 JavaScript 덕분입니다.
HTML은 뼈대, CSS는 인테리어, JavaScript는 전기·설비 시스템입니다.
매장의 운영 시스템입니다. POS기를 누르면 주문이 들어가고, 주방 디스플레이에 표시되고, 계산이 자동으로 되는 것. 매장이 "작동"하게 만드는 모든 것이 JavaScript입니다.
실제 코드 예시 — 포획시장 자동 계산
// 슬라이더를 움직이면 이 계산이 실행됩니다 let 상주인원 = 20458; let 출근율 = 0.85; let 외식율 = 0.70; let 일일수요 = 상주인원 * 출근율 * 외식율; // → 12,174명
관련 용어: Chart.js, CDN
4
태그 (Tag)HTML Element
이해하면 좋아요
▶
HTML에서 내용을 감싸는 표시 기호입니다. <div> <p> <button> 등이 모두 태그입니다.
여는 태그 <p>와 닫는 태그 </p>가 한 쌍으로 내용을 감쌉니다.
메뉴판의 카테고리 구분선입니다. "==전채요리==", "==메인==", "==디저트==" 처럼 내용을 구분하고 의미를 부여합니다.
자주 쓰는 태그
<div> → 영역 나누기 (가장 많이 쓰임) <h1> → 큰 제목 <p> → 문단 (paragraph) <table> → 표 <input> → 입력창 (슬라이더, 텍스트박스 등) <canvas>→ 차트를 그리는 영역
5
URLUniform Resource Locator
꼭 알아야
▶
웹페이지의 주소입니다. 브라우저 상단에 보이는 https://example.com/page 형태.
우리 홈페이지 완성 후 접속할 주소도 URL입니다: https://사용자명.github.io/entry-review/
매장의 실제 주소입니다. "경기도 성남시 수정구 금토동 714 트라이타워 1층"처럼, 인터넷에서 우리 홈페이지를 찾아가는 길입니다.
URL 구조 분석
https://gregoriojong.github.io/entry-review/ │ │ │ │ │ │ │ └─ 페이지 경로 │ │ └─ 저장소 이름 │ └─ GitHub 사용자명 └─ 보안 연결 (자물쇠 표시)

참고만 하세요

이해하면 좋아요

9
레이아웃Layout (Flex / Grid)
이해하면 좋아요
▶
웹페이지에서 각 요소를 어디에 배치할지 정하는 방법입니다.
Flex는 한 줄로 나란히 배치할 때, Grid는 격자(표) 형태로 배치할 때 사용합니다.
매장의 좌석 배치입니다. Flex는 바(bar) 좌석처럼 한 줄로, Grid는 홀 좌석처럼 행/열 격자로 배치하는 것입니다.

10
CDNContent Delivery Network
참고만 하세요
▶
다른 사람이 만든 코드(라이브러리)를 인터넷에서 바로 가져다 쓰는 방법입니다. Chart.js 같은 차트 라이브러리를 직접 만들 필요 없이, CDN 주소 한 줄로 불러옵니다.
식재료 유통업체와 같습니다. 양파를 직접 농사지을 필요 없이, 유통업체(CDN)에서 바로 배달받아 요리에 사용하는 것입니다.
CDN 사용 예시
<!-- Chart.js를 CDN으로 불러오기 --> <script src="https://cdnjs.cloudflare.com/ajax/libs/Chart.js/4.4.1/chart.umd.js"></script> → 이 한 줄로 차트 기능을 통째로 사용 가능!

동작 & 기능5개 용어
12
이벤트Event (click, input, hover)
이해하면 좋아요
▶
사용자가 웹페이지에서 어떤 행동을 했을 때 발생하는 신호입니다.
"클릭했다", "슬라이더를 움직였다", "마우스를 올렸다" 등이 모두 이벤트입니다.
매장의 주문 벨과 같습니다. 손님이 벨을 누르면(이벤트 발생) → 직원이 와서 주문을 받습니다(기능 실행). 벨을 안 누르면 아무 일도 안 일어납니다.
이벤트 예시
<!-- "분석 실행" 버튼을 클릭하면 run() 함수 실행 --> <button onclick="run()">분석 실행</button> <!-- 슬라이더를 움직이면 값 표시 업데이트 --> <input type="range" oninput="updateValue(this)">
13
함수Function
이해하면 좋아요
▶
특정 작업을 수행하는 코드 묶음에 이름을 붙인 것입니다. run(), calculateBEP()처럼 이름으로 불러서 실행합니다.
한번 만들어두면 몇 번이든 재사용할 수 있습니다.
표준화된 조리 레시피입니다. "까르보나라 만들기"라는 레시피(함수)를 한번 정해두면, 주문이 올 때마다 그 레시피를 실행하면 됩니다. 매번 새로 고민할 필요 없습니다.
함수 예시 — BEP 계산
function calculateBEP(고정비, 변동비율) { return 고정비 / (1 - 변동비율); } // 사용: 버튼 클릭 시 호출 let bep = calculateBEP(10490000, 0.65); // → 29,971,429원 (BEP 매출)
14
DOMDocument Object Model
참고만 하세요
▶
JavaScript가 HTML 요소를 찾고 바꿀 수 있게 해주는 구조입니다.
"종합점수 표시 영역을 찾아서 67점으로 바꿔라" 같은 명령이 가능한 이유입니다.
매장의 전광판 제어 시스템입니다. "오늘의 메뉴" 전광판(HTML 요소)을 찾아서 내용을 바꾸는(JavaScript로 수정) 리모컨 같은 것입니다.
DOM 조작 예시
// id가 "score"인 요소를 찾아서 내용 변경 document.getElementById("score").textContent = "67점";
15
fetch / API 호출데이터 가져오기
이해하면 좋아요
▶
웹페이지에서 다른 곳에 있는 데이터를 가져오는 방법입니다.
우리 홈페이지에서 data/triatower.json 파일을 읽어오는 것이 fetch입니다.
API는 "데이터를 주고받는 약속된 방법"입니다.
식재료 발주와 같습니다. 발주서(요청)를 보내면 → 업체가 식재료(데이터)를 배달해줍니다. fetch는 발주 행위, API는 발주 시스템(전화번호, 주문 양식 등)입니다.
fetch 예시
// 트라이타워 분석 데이터를 가져옴 fetch('data/sites/triatower.json') .then(res => res.json()) .then(data => { // data.종합점수 → 67 showDashboard(data); });

Git버전 관리 시스템
꼭 알아야
▶
파일의 모든 변경 이력을 자동으로 기록해주는 시스템입니다.
"언제, 무엇을, 왜 바꿨는지"가 전부 기록되고, 실수하면 이전 버전으로 되돌릴 수 있습니다.
레시피의 개정 이력 노트입니다. "4/14: 소금 양 2g→3g으로 변경 (손님 피드백)", "4/16: 파스타 면 종류 변경". 모든 변경이 기록되고, 잘못 바꿨으면 이전 레시피로 돌아갈 수 있습니다.
18
GitHub깃허브 — 온라인 코드 저장소
꼭 알아야
▶
Git으로 관리하는 파일을 인터넷(클라우드)에 보관하는 서비스입니다. 무료입니다.
컴퓨터가 고장나도 GitHub에 올려둔 파일은 안전하고, 어디서든 접근 가능합니다.
GitHub Pages 기능으로 웹사이트도 무료 호스팅합니다.
레시피 노트를 보관하는 클라우드 금고입니다. 매장(내 컴퓨터)에 화재가 나도 금고(GitHub)에 백업이 있으니 안전합니다. 다른 매장(다른 컴퓨터)에서도 열 수 있습니다.
19
저장소 (Repository)줄여서 Repo
꼭 알아야
▶
하나의 프로젝트에 속하는 모든 파일을 담는 폴더입니다. GitHub에서의 기본 단위입니다.
우리는 entry-review라는 이름의 저장소를 만들 예정입니다.
하나의 매장 운영 바인더와 같습니다. 트라이타워 입점 관련 모든 서류(분석 자료, 계약서, 매출 데이터)가 하나의 바인더(저장소)에 정리되어 있는 것입니다.
20
커밋 (Commit)변경사항 저장
꼭 알아야
▶
변경한 내용을 "이 상태를 기록해둬"라고 확정하는 행위입니다.
커밋할 때 "무엇을 왜 바꿨는지" 메시지를 함께 남깁니다. 이것이 변경 이력이 됩니다.
